### 1. 爬楼梯（基础入门）

**题目描述**
假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。每次你可以爬 `1` 或 `2` 个台阶。你有多少种不同的方法可以爬到楼顶呢？

**输入格式**
一个整数 `n`，表示台阶总数。（1 ≤ n ≤ 45）

**输出格式**
一个整数，表示爬到楼顶的方法数。

**示例**

```text
输入：2
输出：2
解释：有两种方法：1 阶 + 1 阶；2 阶
```

**提示**
斐波那契数列型DP，`dp[i] = dp[i-1] + dp[i-2]`。

### 2. 0-1 背包问题（经典背包）

**题目描述**
有 `N` 件物品和一个容量为 `V` 的背包。每件物品只能使用一次。第 `i` 件物品的体积是 `w[i]`，价值是 `v[i]`。求解将哪些物品装入背包，可使这些物品的总体积不超过背包容量，且总价值最大。

**输入格式**
第一行两个整数 `N, V`，分别表示物品数量和背包容量。
接下来 `N` 行，每行两个整数 `w[i], v[i]`，表示第 `i` 件物品的体积和价值。

**输出格式**
一个整数，表示最大价值。

**示例**

```text
输入：
4 5
1 2
2 4
3 4
4 5
输出：
8
解释：选第1、2、4件物品，总体积1+2+4=7>5？不对，再算：1+2+3=6>5，1+3+4=8>5，2+3+4=9>5，1+2+4=7>5，1+3+4=8>5，2+3+4=9>5，1+2+3=6>5，1+4=5恰好，价值2+5=7；2+3=5价值4+4=8；所以最大8。
```

（注意示例中的数据需自行验证，这里只是示意）

**提示**
二维 `dp[i][j]` 表示前 `i` 件物品放入容量 `j` 的背包的最大价值，可优化为一维。

### 3. 最长递增子序列（LIS）

**题目描述**
给你一个整数数组 `nums`，找到其中最长严格递增子序列的长度。
子序列是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。

**输入格式**
第一行一个整数 `n`，表示数组长度。
第二行 `n` 个整数，表示数组元素。

**输出格式**
一个整数，表示最长递增子序列的长度。

**示例**

```text
输入：
8
10 9 2 5 3 7 101 18
输出：
4
解释：最长递增子序列是 [2,3,7,101] 或 [2,5,7,101] 等，长度为 4。
```

**提示**
朴素 DP 时间复杂度 O(n²)，进阶可用二分优化到 O(n log n)。

### 4. 编辑距离（Levenshtein distance）

**题目描述**
给你两个单词 `word1` 和 `word2`，请计算将 `word1` 转换成 `word2` 所使用的最少操作数。
你可以对一个单词进行如下三种操作：

- 插入一个字符
- 删除一个字符
- 替换一个字符

**输入格式**
两行，每行一个字符串，仅包含小写字母。

**输出格式**
一个整数，表示最少操作步数。

**示例**

```text
输入：
horse
ros
输出：
3
解释：
horse -> rorse (替换 'h' 为 'r')
rorse -> rose (删除 'r')
rose -> ros (删除 'e')
```

**提示**
二维 DP，`dp[i][j]` 表示 `word1[0..i-1]` 到 `word2[0..j-1]` 的最小编辑距离。

### 5. 石子合并（区间DP）

**题目描述**
有 `N` 堆石子排成一排，每堆石子有一定数量。现要将石子有次序地合并成一堆。规定每次只能选相邻的 2 堆合并成新的一堆，并将新的一堆的石子数记为该次合并的得分。
请计算将这 `N` 堆石子合并成一堆的最小得分和最大得分。

**输入格式**
第一行一个整数 `N`（1 ≤ N ≤ 300）。
第二行 `N` 个整数，表示每堆石子的数量。

**输出格式**
一行两个整数，分别表示最小得分和最大得分，中间用空格隔开。

**示例**

```text
输入：
4
4 5 9 4
输出：
44 54
解释：最小得分合并方式：(4+5)=9, (9+9)=18, (18+4)=22 → 9+18+22=49? 实际计算可能有误。典型答案是44和54，需自己验证区间DP。
```

（建议自行推导）

**提示**
区间DP，`dp[i][j]` 表示合并区间 `[i,j]` 的最值，枚举分割点。需预处理前缀和。

### 6. 打家劫舍 III（树形DP）

**题目描述**
小偷又发现了一个新的可行窃的地区。这个地区只有一个入口，称为“根”。除了根之外，每栋房子有且只有一个“父”房子与之相连。一番侦察之后，聪明的小偷意识到“这个地方的所有房屋的排列类似于一棵二叉树”。如果两个直接相连的房子在同一天晚上被打劫，房屋将自动报警。
给定一棵二叉树的根节点 `root`，返回在不触动警报的情况下，小偷能够盗取的最高金额。

**输入格式**
二叉树的层序遍历序列，用 `null` 表示空节点。（具体输入格式可根据需要调整，这里不严格限制）

**示例**

```text
输入: [3,2,3,null,3,null,1]
     3
    / \
   2   3
    \   \
     3   1
输出: 7 
解释: 小偷一次盗窃 3 (根) + 3 (右孩子的右孩子) + 1 (左孩子的右孩子?) 不对，应该是 3 (根) + 3 (左孩子的右孩子) + 1 (右孩子的右孩子) = 7? 但左孩子的右孩子是3，右孩子的右孩子是1，合计7；但根与孩子不能同时选，所以更优是选左孩子的右孩子3+右孩子的右孩子1+根？不能同时选，所以只能选根+孙子？根(3) + 左孩子的右孩子(3) + 右孩子的右孩子(1) = 7，根不与孩子同时选，所以合法。另一种选法：左孩子2+右孩子3+右孩子的右孩子1? 2+3+1=6，小于7。所以最大为7。
```

**提示**
树形DP，每个节点返回两个状态：选择该节点与不选该节点的最大收益。
