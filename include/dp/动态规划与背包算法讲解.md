动态规划（Dynamic Programming, DP）是一种用于求解最优化问题的算法思想，它通过将原问题分解为相互重叠的子问题，并利用子问题的最优解构造原问题的最优解。背包问题是动态规划的经典应用场景，下面将系统讲解动态规划的基本原理，并以0/1背包和完全背包为例，展示其具体实现与变体。

---

## 一、动态规划核心思想

动态规划适用于具有以下特征的问题：

1. **最优子结构**：问题的最优解包含子问题的最优解。
2. **重叠子问题**：不同子问题会重复出现，而非每次都是新问题。
3. **无后效性**：某阶段的状态一旦确定，则此后过程的演变不再受此前状态的影响。

**基本步骤**：

1. **定义状态**：用一组参数描述问题所处的阶段与约束，例如 `dp[i][j]`。
2. **建立状态转移方程**：给出状态之间的递推关系。
3. **初始化边界条件**：确定最小子问题的解。
4. **确定计算顺序**：通常自底向上迭代，确保子问题先于父问题求解。

例如，斐波那契数列：定义 `dp[n]` 为第 `n` 项的值，转移方程 `dp[n] = dp[n-1] + dp[n-2]`，边界 `dp[0]=0, dp[1]=1`，按 `n` 递增计算即可。

## 二、背包问题概述

背包问题泛指：给定一组物品，每种物品有重量和价值，在限定的总重量内，如何选择物品使得总价值最大。根据物品的选取限制，衍生出多种变体，其中最常见的是 **0/1背包** 和 **完全背包**。

---

## 三、0/1背包问题

### 1. 问题描述

有 `N` 件物品，第 `i` 件物品重量为 `w[i]`，价值为 `v[i]`。背包容量为 `C`。每件物品最多选一次，求能装入背包的最大总价值。

### 2. 动态规划解法

**状态定义**：
设 `dp[i][j]` 表示从前 `i` 件物品中选出若干件，放入容量为 `j` 的背包所能获得的最大价值。
其中 `i` 的取值范围 `0 ~ N`，`j` 的取值范围 `0 ~ C`。

**状态转移方程**：
考虑第 `i` 件物品（下标从1开始）：

- 如果不选第 `i` 件，则价值等于 `dp[i-1][j]`。
- 如果选第 `i` 件（前提 `j ≥ w[i]`），则价值等于 `dp[i-1][j-w[i]] + v[i]`。

因此：

```text
dp[i][j] = 
    dp[i-1][j]                          (j < w[i])
    max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])   (j ≥ w[i])
```

**初始化**：
`dp[0][j] = 0`，表示没有物品时，任何容量下的价值为0。

**计算顺序**：
按 `i` 从1到N递增，对每个 `i`，按 `j` 从0到C递增（或递减均可，但后续空间优化有讲究）。

**示例**：
`N=3, C=5`，物品 (w,v) 分别为 (2,3)、(3,4)、(4,5)。
手动模拟可得最大价值为 7（选物品1和2）。

**复杂度**：时间 `O(N*C)`，空间 `O(N*C)`。

### 3. 空间优化

观察到 `dp[i][j]` 只与 `dp[i-1][...]` 有关，可用一维数组 `dp[j]` 滚动更新。但必须注意 **内层循环 j 从 C 递减到 w[i]**，以保证每个物品只被考虑一次（因为正序会导致同一物品被多次选取）。

```plaintext
for i = 1 to N
    for j = C down to w[i]
        dp[j] = max(dp[j], dp[j-w[i]] + v[i])
```

最终 `dp[C]` 即为答案。

## 四、完全背包问题

### 1. 问题描述

与0/1背包类似，但 **每种物品可以无限次选取**。

### 2. 动态规划解法

**状态定义**：仍用 `dp[i][j]` 表示前 `i` 件物品在容量 `j` 下的最大价值。

**状态转移方程**：
对于第 `i` 件物品，可以选0次、1次、2次……直到容量不够。最直接的想法是枚举选取次数 `k`：

```text
dp[i][j] = max{ dp[i-1][j - k*w[i]] + k*v[i] }   (0 ≤ k ≤ j/w[i])
```

但这会引入三层循环，效率低。观察发现，完全背包与0/1背包的转移差异在于：当决定选取第 `i` 件物品后，我们仍可以继续选它，因此子问题不是 `dp[i-1][j-w[i]]`，而是 `dp[i][j-w[i]]`（允许在同一层继续选）。于是得到优化后的转移方程：

```text
dp[i][j] = max( dp[i-1][j], dp[i][j - w[i]] + v[i] )   (j ≥ w[i])
```

**解释**：要么不选第 `i` 件，继承上一行结果；要么选一件（即从当前行容量减少 `w[i]` 的状态转移过来，因为选完一件后还可以继续选）。

**初始化**：`dp[0][j]=0`。

**计算顺序**：按 `i` 递增，对每个 `i`，按 `j` 从0到C递增（保证 `dp[i][j-w[i]]` 已更新）。

### 3. 空间优化

同样可用一维数组，此时 **内层循环 j 从 w[i] 到 C 正序** 即可：

```plaintext
for i = 1 to N
    for j = w[i] to C
        dp[j] = max(dp[j], dp[j-w[i]] + v[i])
```

正序保证了同一物品可以被多次选取，因为 `dp[j-w[i]]` 可能已经包含了当前物品的选取。

**示例**：
同上数据，完全背包下最大价值为多少？可以反复选物品1（重量2价值3），容量5可装2个物品1再加物品2？但物品2重量3价值4，组合为2+3=5，价值3+3+4=10；或选两个物品1和一个物品1？实际上两个物品1（重量4价值6）再加一个物品1（重量2）不行，容量5最优是选两个物品1（重量4价值6）加？其实物品3重量4价值5，两个物品1价值6比一个物品3价值5高，所以最大可能是选两个物品1（价值6）加一个？容量5选两个物品1（4）剩余1不能加任何物品，总价值6；但选物品1+物品2=2+3=5，价值3+4=7；选物品3=4，剩余1，价值5；选物品2+？物品2重量3剩余2可以再选一个物品1，总价值4+3=7。所以最优是7？等一下，完全背包允许无限次，物品1价值重量比1.5，物品2约1.33，物品3=1.25，显然物品1最好，尽量多选物品1，容量5最多选2个物品1（重量4），价值6，剩余1无法加任何物品，所以最大是6？但上面物品1+物品2=7大于6，所以最优是7。说明完全背包下，最优解是物品1和物品2各一，价值7。实际上物品1选两个是6，选一个+物品2是7，选两个物品1+？不行。所以答案是7。

## 五、其他常见背包变体

1. **多重背包**：每种物品有有限数量 `c[i]` 个。解法可将每种物品拆成 `c[i]` 个独立的0/1物品（复杂度高），或用二进制优化（将数量拆成2的幂次组合），或用单调队列优化。
2. **混合背包**：部分物品只能取一次，部分可以无限次，部分有限次。只需在循环内根据类型选择不同的内层循环方向。
3. **二维费用背包**：物品有两种代价（如重量和体积），背包有两种容量限制。状态扩展为 `dp[i][j][k]`，通常可压缩一维。
4. **分组背包**：物品分为若干组，每组内最多选一个。状态转移时，对于每组，考虑选组内某个物品或不选该组。

---

## 六、总结

动态规划通过定义状态和转移方程，将复杂问题转化为子问题的递推求解。背包问题是理解DP的绝佳范例：

- **0/1背包** 的核心是“选或不选”，一维实现时需逆序更新。
- **完全背包** 的核心是“可以重复选”，一维实现时需正序更新。
- 其他变体则是在此基础上的扩展，核心思想仍是通过合理设计状态和转移来适应不同约束。

掌握动态规划需要多练习，从经典问题中体会“状态”和“转移”的构建方法。背包问题作为入门经典，能帮助初学者快速建立DP思维。
